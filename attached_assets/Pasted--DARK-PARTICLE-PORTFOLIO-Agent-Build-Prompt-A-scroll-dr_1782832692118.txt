# DARK PARTICLE PORTFOLIO — Agent Build Prompt
## A scroll-driven, WebGL-particle, dark-themed developer portfolio (Dala/Unseen Studio inspired motion language — original content & shape)

> **Instructions for the agent:** This is a complete build specification for Replit.
> Build a production-quality, dark-themed developer portfolio with a 3D particle hero,
> scroll-scrubbed animation, and pinned section storytelling — in the visual language of
> premium agency sites (Unseen Studio, Locomotive-style smooth scroll), but with 100%

## 0. Reference Direction (style language only — not 1:1 copy)

The target *feel*: a near-black canvas where a 3D particle formation sits behind/within the
hero, gently rotating and breathing on idle, then **morphing shape as the user scrolls** —
particles flow from one formation into the next, synced precisely to scroll position rather
than time. Sections are minimal, full-bleed, generously spaced, with large confident type
and a loading counter on first paint. This is restrained, premium, technical — not flashy
or maximalist. Think: a control room for someone's career, not a flyer.

**What we are NOT copying:** Dala's literal brain mesh, their exact copy, their logo, their
color of accent. We are copying the *system*: particle field + scroll-scrubbed morph +
counted boot + pinned section reveals + dark minimal type-led layout.

## 4. The Particle System (Core Signature Element)

This is the single most important piece — get this right and everything else can be simpler.

### 4.1 Setup

```jsx
// ParticleField.jsx
import { Canvas, useFrame } from '@react-three/fiber';
import { useRef, useMemo } from 'react';
import * as THREE from 'three';

const PARTICLE_COUNT = 4000;

// Define 4 formation states as point clouds — sphere, torus, scattered cube, DNA-helix-like spiral
function generateFormation(type, count) {
  const positions = new Float32Array(count * 3);
  for (let i = 0; i < count; i++) {
    let x, y, z;
    if (type === 'sphere') {
      const phi = Math.acos(-1 + (2 * i) / count);
      const theta = Math.sqrt(count * Math.PI) * phi;
      const r = 3.2;
      x = r * Math.cos(theta) * Math.sin(phi);
      y = r * Math.sin(theta) * Math.sin(phi);
      z = r * Math.cos(phi);
    } else if (type === 'torus') {
      const u = Math.random() * Math.PI * 2;
      const v = Math.random() * Math.PI * 2;
      const R = 2.8, r = 1.1;
      x = (R + r * Math.cos(v)) * Math.cos(u);
      y = (R + r * Math.cos(v)) * Math.sin(u);
      z = r * Math.sin(v);
    } else if (type === 'grid') {
      const side = Math.cbrt(count);
      const idx = i % Math.pow(Math.ceil(side), 3);
      x = ((idx % side) / side - 0.5) * 6;
      y = (Math.floor(idx / side) % side / side - 0.5) * 6;
      z = (Math.floor(idx / (side * side)) / side - 0.5) * 6;
    } else { // helix
      const t = (i / count) * Math.PI * 8;
      const r = 2.5;
      x = r * Math.cos(t);
      y = (i / count) * 8 - 4;
      z = r * Math.sin(t);
    }
    positions[i * 3] = x;
    positions[i * 3 + 1] = y;
    positions[i * 3 + 2] = z;
  }
  return positions;
}
```

**Formation map (4 states, one per major scroll section):**
1. `sphere` — Hero (idle, slow rotation)
2. `helix` — Journey/Experience section (suggests a path/timeline)
3. `grid` — Skills section (suggests structure/system)
4. `torus` — Contact/closing section (suggests completion/loop)

### 4.2 Scroll-Driven Morph (the signature mechanic)

```jsx
function ParticleMesh({ scrollProgress }) {
  const meshRef = useRef();
  const formations = useMemo(() => ({
    sphere: generateFormation('sphere', PARTICLE_COUNT),
    helix: generateFormation('helix', PARTICLE_COUNT),
    grid: generateFormation('grid', PARTICLE_COUNT),
    torus: generateFormation('torus', PARTICLE_COUNT),
  }), []);

  const currentPositions = useRef(new Float32Array(PARTICLE_COUNT * 3));

  useFrame((state) => {
    // Determine which two formations to interpolate between based on scrollProgress (0–1)
    const stages = ['sphere', 'helix', 'grid', 'torus'];
    const stageProgress = scrollProgress.current * (stages.length - 1);
    const stageIndex = Math.min(Math.floor(stageProgress), stages.length - 2);
    const localT = stageProgress - stageIndex;

    const from = formations[stages[stageIndex]];
    const to = formations[stages[stageIndex + 1]];

    for (let i = 0; i < PARTICLE_COUNT * 3; i++) {
      // Ease the interpolation — smoothstep
      const eased = localT * localT * (3 - 2 * localT);
      currentPositions.current[i] = from[i] + (to[i] - from[i]) * eased;
    }

    // Gentle idle rotation always applied on top
    meshRef.current.rotation.y = state.clock.elapsedTime * 0.05;
    meshRef.current.geometry.attributes.position.array = currentPositions.current;
    meshRef.current.geometry.attributes.position.needsUpdate = true;
  });

  return (
    <points ref={meshRef}>
      <bufferGeometry>
        <bufferAttribute attach="attributes-position" count={PARTICLE_COUNT} array={currentPositions.current} itemSize={3} />
      </bufferGeometry>
      <pointsMaterial size={0.025} color="#d4ff4f" sizeAttenuation transparent opacity={0.85} />
    </points>
  );
}
```

### 4.3 Wiring Scroll Progress (GSAP ScrollTrigger)

```jsx
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
gsap.registerPlugin(ScrollTrigger);

// In the main App component, after mount:
const scrollProgress = useRef(0);

useEffect(() => {
  ScrollTrigger.create({
    trigger: document.body,
    start: 'top top',
    end: 'bottom bottom',
    scrub: 1, // 1 second smoothing lag — gives the "fluid" feel, never raw 1:1
    onUpdate: (self) => {
      scrollProgress.current = self.progress; // 0 to 1 across full page
    },
  });
}, []);
```

Pass `scrollProgress` ref into the `<Canvas>` particle component as a prop (not state — using a
ref avoids re-render thrash on every scroll tick, which is critical for 60fps).

**Particle field positioning:** Fixed full-viewport background (`position: fixed; z-index: 0`),
sits behind all content sections (`z-index: 1+`), with a subtle radial dark vignette overlay so
particles fade near content edges and never compete with text legibility.

### 4.4 Idle Behavior

When `scrollProgress` is not actively changing (user paused), particles should still feel alive:
add a small per-particle sine-wave jitter (`+= Math.sin(time + i) * 0.002`) on top of the morphed
position, so the field breathes even at rest. This is what separates "premium" from "static."

## 5. Loading Sequence

On first mount, before the particle field and content are visible:

```
┌─────────────────────────────┐
│                              │
│         [name initials]      │
│                              │
│           ENTERING           │
│            47%                │  ← counts 0→100 over ~1.8s
│      ▬▬▬▬▬▬▬▬░░░░░░░          │  ← thin lime progress bar
│                              │
└─────────────────────────────┘
```

- Pure black screen, centered content
- Counter increments via `requestAnimationFrame`, easing out near the end (fast start, settle at 100)
- On reaching 100%: counter fades, thin horizontal line expands to full width then splits/fades,
  revealing the hero underneath with a 400ms crossfade
- Total duration: 1.8–2.2s, **never longer** — this is a taste signal, not a delay tactic
- Respect `prefers-reduced-motion`: skip straight to content with no counter if set
- Store a `sessionStorage` flag so the loader only plays once per session, not on every internal nav

## 7. Animation Reference Table

| Event | Technique | Timing |
|---|---|---|
| Page load counter | `requestAnimationFrame` increment | 1.8–2.2s |
| Loader → hero crossfade | CSS opacity transition | 400ms |
| Particle formation morph | Scroll-scrubbed lerp (Section 4.3) | tied to scroll, `scrub: 1` |
| Particle idle jitter | Per-frame sine offset | continuous, subtle |
| Section reveal-on-enter | GSAP fade + translateY(24px→0) | 700ms, `power2.out`, triggered at `top 80%` |
| Stat counter count-up | GSAP `onUpdate` numeric tween | 1.2s, `power2.out` |
| Timeline line draw | `stroke-dashoffset` scrubbed to scroll | tied to scroll within section |
| Process line fill | `scaleX` scrubbed to scroll | tied to scroll within section |
| Project image hover | CSS `transform: scale()` | 600ms ease |
| Skill icon hover | CSS `filter: grayscale()` toggle | 300ms |
| Nav background on scroll | CSS `backdrop-filter` + opacity | 300ms, triggered past hero height |
| Testimonial rotation | CSS opacity crossfade | 6s interval, 500ms transition |

**Critical performance rule:** only animate `transform` and `opacity` via CSS/GSAP for anything
outside the particle canvas — never animate `width`, `height`, `top`, `left` directly, this kills
frame rate alongside a live WebGL canvas running simultaneously.

## 8. Accessibility & Responsiveness

- `prefers-reduced-motion: reduce` →
  - Skip loading counter entirely
  - Particle field still renders but freezes formation at `sphere` (no scroll-morph), only keeps
    minimal idle rotation
  - All section reveal animations become instant (`opacity: 1`, no transform delay)
- Keyboard navigation: all CTAs, nav links, and project cards are real `<a>`/`<button>` elements,
  visible focus ring in `--accent` color
- Color contrast: `--text-secondary` on `--bg-base` must pass WCAG AA for body text (verify ratio,
  adjust lightness if needed — do not ship a contrast-failing gray)
- Mobile (`< 768px`):
  - Particle canvas renders at reduced `PARTICLE_COUNT` (1200 instead of 4000) for performance
  - Hero type scales down via `clamp()`, nav collapses to hamburger
  - Timeline becomes single-column, process steps stack vertically
  - Disable the scroll-scrubbed morph's `scrub` smoothing lag on mobile (set `scrub: true` instead
    of `scrub: 1`) since touch-scroll physics differ from desktop wheel scroll
